# TECNOLOGÍA LEXIA V1

## Objetivo

Definir la tecnología inicial para construir la primera versión funcional de LEXIA.

---

# PRINCIPIO GENERAL

LEXIA V1 deberá construirse de forma simple, escalable y segura.

La prioridad será validar el modelo de negocio antes de realizar una inversión tecnológica compleja.

---

# FRONTEND

## Función

Interfaz visual utilizada por usuarios, abogados y administradores.

## Tecnología sugerida

- React.
- Next.js.
- Tailwind CSS.

## Pantallas principales

- Inicio.
- Chat LEXIA.
- Diagnóstico jurídico.
- Expediente.
- Abogados.
- LEXIA PAY.
- Seguimiento.

---

# BACKEND

## Función

Procesar información, administrar usuarios, expedientes, pagos y comunicaciones.

## Tecnología sugerida

- Node.js.
- Express.
- NestJS.

## Funciones principales

- Registro de usuarios.
- Gestión de expedientes.
- Gestión de abogados.
- Gestión de documentos.
- Gestión de pagos.
- Conexión con IA.

---

# BASE DE DATOS

## Función

Guardar información estructurada de la plataforma.

## Tecnología sugerida

- PostgreSQL.

## Información a almacenar

- Usuarios.
- Abogados.
- Expedientes.
- Documentos.
- Pagos.
- Hitos.
- Actividades.
- Mensajes.

---

# INTELIGENCIA ARTIFICIAL

## Función

Analizar consultas, clasificar problemas legales y generar orientación inicial.

## Capacidades

- Clasificación jurídica.
- Entrevista guiada.
- Diagnóstico preliminar.
- Generación documental.
- Resumen de expedientes.

---

# ALMACENAMIENTO DE DOCUMENTOS

## Función

Guardar archivos cargados por usuarios y abogados.

## Tipos de archivo

- PDF.
- Imagen.
- Word.

## Requisitos

- Seguridad.
- Control de acceso.
- Vinculación con expediente.

---

# AUTENTICACIÓN

## Función

Permitir acceso seguro a la plataforma.

## Métodos iniciales

- Correo electrónico.
- Contraseña.

## Métodos futuros

- Google.
- Apple.
- Firma electrónica.

---

# ROLES DEL SISTEMA

## Usuario

Persona que solicita orientación jurídica.

## Abogado

Profesional que atiende casos.

## Administrador

Persona encargada de validar abogados, supervisar expedientes y gestionar operaciones.

---

# SEGURIDAD

## Requisitos básicos

- Contraseñas protegidas.
- Acceso por roles.
- Protección de datos personales.
- Registro de actividades.
- Control de documentos.

---

# INTEGRACIONES FUTURAS

- Pasarela de pagos.
- Firma electrónica.
- WhatsApp.
- Correo electrónico.
- Validación de cédula profesional.
- Videollamadas.
- Calendario de audiencias.

---

# OBJETIVO TÉCNICO DE V1

Construir una versión funcional que permita:

1. Registrar usuarios.
2. Recibir consultas.
3. Clasificar asuntos.
4. Crear expedientes.
5. Subir documentos.
6. Asignar abogados.
7. Registrar avances.
8. Administrar pagos mediante LEXIA PAY.
