# CASOS DE USO LEXIA

## Objetivo

Definir la interacción de los actores principales con la plataforma LEXIA.

---

# ACTORES DEL SISTEMA

1. Usuario.
2. LEXIA IA.
3. Abogado.
4. Administrador.
5. LEXIA PAY.

---

# CASO DE USO CU-01

## Consulta Jurídica Inicial

### Actor principal

Usuario.

### Descripción

El usuario expone su problema legal a LEXIA IA.

### Flujo principal

1. El usuario inicia sesión.
2. Describe su problema.
3. LEXIA IA analiza la consulta.
4. Identifica la materia jurídica.
5. Clasifica el nivel de riesgo.
6. Solicita información complementaria.
7. Genera orientación jurídica inicial.

### Resultado

Usuario orientado jurídicamente.

---

# CASO DE USO CU-02

## Creación de Expediente

### Actor principal

LEXIA IA.

### Descripción

Crear expediente digital para el usuario.

### Flujo principal

1. LEXIA IA identifica un asunto viable.
2. Genera número de expediente.
3. Crea registro digital.
4. Asocia documentos.
5. Registra actividad inicial.

### Resultado

Expediente activo.

---

# CASO DE USO CU-03

## Generación de Documentos

### Actor principal

Usuario.

### Flujo principal

1. Usuario solicita documento.
2. LEXIA IA recopila información.
3. Genera borrador.
4. Usuario revisa.
5. Documento queda almacenado en expediente.

### Resultado

Documento generado.

---

# CASO DE USO CU-04

## Solicitud de Opinión Especializada

### Actor principal

Usuario.

### Flujo principal

1. Usuario solicita revisión profesional.
2. Realiza pago correspondiente.
3. Sistema asigna abogado disponible.
4. Abogado analiza expediente.
5. Emite opinión jurídica.

### Resultado

Opinión especializada entregada.

---

# CASO DE USO CU-05

## Contratación de Abogado

### Actor principal

Usuario.

### Flujo principal

1. Usuario decide contratar abogado.
2. Sistema muestra opciones disponibles.
3. Usuario selecciona abogado.
4. Se genera contrato digital.
5. Se activa LEXIA PAY.

### Resultado

Abogado asignado.

---

# CASO DE USO CU-06

## Administración de Pagos

### Actor principal

LEXIA PAY.

### Flujo principal

1. Cliente realiza pago.
2. Fondos quedan retenidos.
3. Se crean hitos de trabajo.
4. Abogado entrega avances.
5. Cliente valida avance.
6. Sistema libera pago correspondiente.

### Resultado

Pago seguro y trazable.

---

# CASO DE USO CU-07

## Seguimiento del Expediente

### Actor principal

Usuario.

### Flujo principal

1. Usuario accede al expediente.
2. Consulta avances.
3. Revisa documentos.
4. Revisa actividades.
5. Consulta pagos realizados.

### Resultado

Seguimiento transparente.

---

# CASO DE USO CU-08

## Sustitución de Abogado

### Actor principal

Administrador.

### Descripción

Garantizar continuidad del expediente.

### Flujo principal

1. Abogado deja de atender asunto.
2. Sistema detecta inactividad.
3. Administrador revisa expediente.
4. Se asigna nuevo abogado.
5. Nuevo abogado recibe historial completo.

### Resultado

Continuidad jurídica garantizada.

---

# CASO DE USO CU-09

## Calificación de Servicio

### Actor principal

Usuario.

### Flujo principal

1. Concluye el asunto.
2. Usuario evalúa servicio.
3. Sistema registra calificación.
4. Calificación impacta historial profesional.

### Resultado

Control de calidad permanente.

---

# CASO DE USO CU-10

## Cierre de Expediente

### Actor principal

Administrador.

### Flujo principal

1. Asunto concluido.
2. Se verifican pagos.
3. Se genera reporte final.
4. Se archiva expediente.

### Resultado

Expediente cerrado y conservado para consulta futura.

---

# PRINCIPIO RECTOR

Todo caso deberá poder continuar aun cuando cambie el abogado responsable.

El expediente pertenece al usuario.

La información pertenece al expediente.

LEXIA garantiza la continuidad del servicio.
